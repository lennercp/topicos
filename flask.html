Flask

Micro framework web

"Web development, one drop at a time"

INTRODUÇÃO

O desenvolvimento de um site pode ser bem cansativo, principalmente sem o uso de frameworks. Mas quando o seu projeto é pequeno, e está utilizando frameworks, toma-se muito tempo configurando pacotes, banco de dados, entre outros. Esses que no seu projeto nem irá utilizar, com essa intenção que foi criado o Flask, um micro-framework web python que com quase 0 configuração consegue colocar um site no ar.

PRIMEIROS PASSOS

Para inicar o uso do flask deve importar-lo utilizando o comando 

pip install flask

Exemplo de codigo Flask:

from flask import Flask
app = Flask(__name__)

@app.route("/")
def index():
    return 'Hello World!'

if __name__ == "__main__":
    app.run(host='0.0.0.0', port=5000)

@app.route

É um decorador, que recebe um parâmetro sendo uma string, que será a url e quando for acessada vai rodar a função que estará logo abaixo. No caso acima quando for acessada a url "/"", que é a url da pagina inicial, vai mostrar na tela a mensagem "Hello World!"

Outro exemplo de codigo

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    return 'Hello World!'

@app.route('/adulto/<idade>')
def idade(idade):
    return render_template('idade.html', idade=idade)
  • <name> é como se passa parâmetro na url, então se acessar a url idade/18, o '18 vai ser passado para a função abaixo. Obs: pode ser qualquer palavra dentro de <>
  • render_template é a função do Flask para renderizar um template, template é uma página html com alguns "poderes". A função recebe como primeiro parâmetro o nome do template, sendo que deve estar dentro de uma pasta chamada templates, e para enviar dados para do template coloca o nome da variavel e o dado, que no caso acima fica o idade=idade.

idade.html

<!doctype html>
<title>Você é adulto</title>
{% if int(idade) >= 18%}
    <h1>Você é adulto e tem {{ idade }} anos!</h1>
{% else %}
    <h1>Você não é adulto!</h1>
{% endif %}
  • {% %} é a forma como se usa lógica dentro dos templates
  • {{ }} é a forma como se pega parametros que foram passados para dentro do template, no caso a idade

VANTAGENS

Em suma, as ferramentas Git e Github são complementares pois cada uma possui uma vantagem, mas elas não precisam serem usadas necessariamente juntas. A vantagem de se usar o Git é o fato de que não é necessário acesso a algum servidor externo para que se realizem todas as ações, o que faz estas ações serem bem mais rápidas, mas a desvantagem disso é que quaisquer problemas com os repositórios locais ou com a própria máquina local implicam em problemas com o projeto em si. O Github, por sua vez, tem a vantagem de armazenar os repositórios e todas suas informações remotamente, e, sendo assim, não são necessárias preocupações com problemas no ambiente de trabalho local, pois mesmo que hajam problemas, o mesmo repositório pode ser acessado de qualquer outro ambiente e enviado(pull) para um ambiente local novamente. Por outro lado, ele tem a desvantagem de precisar de conexão com um servidor externo e com a internet, o que faz com que, no caso de quaisquer problemas com essas conexões, as ações não possam ser feitas devidamente, o que pode acarretar em percas de trabalho. Percebem? Cada ferramenta compensa as desvantagens da outra, e, por isso, o ideal é serem usadas juntas.
