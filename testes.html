TDD e Pytest

Técnicas de Teste de Software

INTRODUÇÃO

A palavra "teste" é algo que se ouve muito nos dias atuais, mesmo fora da área de desenvolvimento de software. Todos nós, de algum jeito, fazemos ou passamos por algum tipo de teste em nossas vidas. Quando fazemos uma prova, nosso aprendizado está sendo testado por um professor; Quando experimentamos uma roupa, estamos "testando" se ela serve em nós, e se irá combinar com a ocasião que queremos, e por aí em diante.

Que os testes estão presentes em nosso cotidiano todos nós sabemos, mas, e quanto aos testes no contexto do desenvolvimento de software? Para quem desenvolve, sabe que eles são uma parte crucial para absolutamente qualquer projeto bem sucedido, e para quem não desenvolve, ou ainda não desenvolve, vamos apresentar a seguir uma das principais técnicas de teste de software, a fim de mostrar a importância dessa etapa do desenvolvimento.

DEFINIÇÕES

Primeiro, é importante que se diga algo muito importante sobre os testes: Há um grande conflito de interesses envolvido nele. Mas por que isso? Isso se deve ao fato de que quem testa o software é quem o desenvolve, e por isso, quem testa o faz pensando em provar que o seu software funciona, e não em detectar os erros que ele possa ter, o que faz com que o cliente muitas vezes seja quem encontra os erros ao usar esse software. É importante que se diga que há muitos tipos de teste: Testes de Validação de entrada, testes de desempenho, testes de implantação, testes de aceitação, testes de recuperação, testes de segurança, entre muitos outros. Nesse contexto, realizar estes testes sem a mentalidade certa, sem organização ou de maneira errônea pode acarretar em prejuízos enormes para o bom desenvolvimento e manutenção deste software, e é aí que entra a técnica TDD.

Do inglês "Test-driven Development"(Desenvolvimento guiado por testes), a TDD é uma técnica estabelecida para entregar, com mais rapidez e sustentabilidade, um software de maior qualidade e feito mais organizadamente. Esta técnica se baseia em uma ideia simples: escrever os casos de teste antes de escrever o programa em si. Pode-se dizer que ela se trata na verdade de uma técnica de projeto, fundamentada no uso de pequenos testes para projetar o sistema de forma emergente e obter algum valor rápido enquanto criamos um software de confiança. E no contexto do desenvolvimento na linguagem Python, temos uma poderosa biblioteca para nos auxiliar na realização desses testes e no desenvolvimento usando a técnica TDD, a biblioteca Pytest.


Para instalar a biblioteca pytest, basta rodar o seguinte comando: 

pip install -U pytest

Exemplo de código usando a biblioteca pytest: 

                def soma_lista(lista):
                  if type(lista) is list:
                    if len(lista) == 0:
                      return "Lista Vazia"
                    else:
                    soma = sum(lista)
                    return soma
                  else:
                    return "Erro! Apenas listas são permitidas"


                def test_somar_lista_3_valores():
                  assert soma_lista([2,3,4]) == 9

                def test_somar_lista_1_valor():
                  assert soma_lista([2]) == 2

                def test_somar_lista_0_valor():
                  assert soma_lista([]) == 'Lista Vazia'

                def test_somar_sem_ser_lista():
                  assert soma_lista(2) == 'Erro! Apenas listas são permitidas'

assert

A palavra assert significa "assegurar". Se trata de uma função reservada da biblioteca pytest que retorna sucesso apenas quando assegura que uma determinada operação é bem sucedida para todos os dados valores.

Nesse exemplo, este código python possui a função soma_lista, que recebe uma lista e retorna a soma dos valores desta lista; Se a lista for vazia, ele retorna a mensagem "Lista Vazia"; E se o objeto informado não for uma lista, ele retorna uma mensagem de erro. O pytest usa a palavra reservada test para as funções que serão usadas para testar o código, logo todas as funções abaixo que começam com ela são as funções que o pytest irá utilizar nos testes. No caso desse exemplo, existem quatro funções: A primeira testa se a soma de três valores(neste exemplo sendo 2, 3 e 4) terá sempre o mesmo resultado; Na segunda, ele testa a mesma coisa para apenas um valor; Na terceira, ele testa se a funcionalidade de retorno da mensagem no caso da lista vazia está funcionando nos casos em que um valor não é informado; E por último, ele testa se a mensagem de erro para o caso do objeto passado não ser uma lista está funcionando corretamente.

Para fazer os testes, basta rodar no terminal o comando: 

$pytest

E após isso será retornado a porcentagem de sucesso nos testes do programa
